# Writing Guidelines for Image Descriptions

## Voice and Tone

Adapt your voice to the section you are writing:

- **Visual Content**: Use objective, descriptive language. Describe what is present without interpretation. Write in third person.
- **Educational Significance**: Address the reader directly with "you" and "your" to create engagement and connection to learning. Be conversational but authoritative.

Across all sections:
- Prefer active voice, but use passive where it reads more naturally in visual descriptions.
- Eliminate filler words: very, really, just, quite, somewhat, rather, basically.
- Choose precise verbs over adverb-verb combinations.

## Description Quality Principles

### Lead with the Focus
Identify the most important element of the image and begin your description there. This anchors the reader immediately. Do not delay key information behind background details or peripheral elements.

### Be Specific and Concrete
Describe what you observe with precision: "three blue bars increasing in height from 20 to 45 to 78" rather than "a bar chart showing growth." Specificity enables the reader to construct an accurate mental image.

### Balance Completeness and Concision
All description is partial, but should not omit key information. Use an economy of language — every sentence should earn its place.

### Reflect Educational Intention
The image exists to serve a learning purpose. Your description should achieve the same purpose. If an image provides evidence, your description should convey that evidence. If it illustrates a process, your description should make that process clear.

### Order for Efficiency
Place important information first and provide cues about what follows. Avoid constructions that delay key content, such as describing background elements before the main subject.

## Screen Reader Optimisation

Apply these techniques actively throughout your descriptions:

- **Commas for pauses** — separate data elements with commas so screen readers create natural pauses: "State 1, volume equals 0.5 cubic metres, pressure equals 200 kilopascals."
- **"Equals" for data attachment** — attach values to their variables using the word "equals" rather than colons or dashes: "x equals 3, y equals 7" not "x: 3, y: 7." This is clearer when read aloud.
- **Number sequences** — when listing multiple elements, number them explicitly: "Label 1, Q, positioned at the left. Label 2, P.V, positioned at the upper right. Label 3, U, positioned at the lower right."
- **Consistent labelling** — name each element and use that name consistently. Do not switch between "the curve," "the line," and "the plot" for the same element.

## British English Requirements

All content must use British English:

- **Use -ise/-yse endings**: specialise, analyse, organise, recognise.
- **Use British spelling**: colour, centre, programme, catalogue, travelled.
- **Numbers**: write out one through nine as words; use digits for 10 and above; always use digits with units (3 km, 5 ml).
- **Write "per cent"** in text; use the % symbol only in tables.
- **Oxford comma**: always use in lists of three or more items.

## Disability Language

- **Use identity-first language**: "disabled people" not "people with disabilities."
- **Use neutral terms**: "wheelchair user" not "wheelchair-bound."
- **Never use**: suffers from, afflicted with, victim of, confined to.
- **Focus on barriers**, not individual impairments, following the Social Model of Disability.

## Grammar Essentials

- Use "that" for defining clauses (no commas); use "which" for non-defining clauses (with commas).
- Use en dash (–) without spaces for ranges: 2020–2024.
- Use hyphens for compound modifiers: up-to-date information.
